# 미국주식 개장 90분 FVG 자동매매 봇 설계

## 1. 목표

유튜브 쇼츠 `rqN9WV_lyZs`의 내용을 바탕으로, 미국 정규장 개장 직후 90분 동안만 동작하는 자동매매 봇을 설계한다.

이 문서는 "영상에서 말한 감각적 규칙"을 "기계가 실행 가능한 명세"로 바꾸는 것이 목적이다. 수익 보장은 포함하지 않으며, 실거래 전에는 백테스트와 모의투자가 필수다.

## 2. 핵심 전제

- 시장: 미국 정규장 주식
- 기준 시간대: `America/New_York`
- 감시 시간: `09:30:00` ~ `11:00:00 ET`
- 기본 전략 방향: `롱 전용`
- 기본 체결 단위: `5분봉`, `15분봉`
- 1일 최대 진입 횟수: `1회`

영상 해석상 핵심은 다음과 같다.

- `09:30~09:45` 첫 15분봉의 고가/저가를 기준선으로 사용
- 이후 5분봉에서 기준선 몸통 돌파 + FVG 형성을 확인
- FVG 되돌림 구간에서 진입
- 손절은 FVG 형성 직전 구조 저점
- 손익비는 `1:2` 또는 `1:3`

## 3. V1 권장 범위

처음부터 "모든 미국주식"을 대상으로 만들지 않는다. V1은 아래처럼 제한한다.

- 종목: `SPY` 단일 종목
- 브로커: 어댑터 구조로 분리
- 데이터 해상도: `1분봉 원천 데이터`를 받아 내부에서 `5분/15분` 리샘플링
- 주문 유형: `지정가 진입`, `스탑 손절`, `리밋 익절`

이유는 단순하다.

- 개장 직후 단일 종목 전략은 유동성, 뉴스, 체결 오차 영향을 크게 받는다.
- 개별 종목은 실적 발표, 갭, 거래정지, LULD 이슈가 많다.
- `SPY`는 유동성이 매우 높아 백테스트와 실거래의 괴리가 상대적으로 작다.

V2부터 `QQQ`, `AAPL`, `MSFT`, `NVDA`, `META`, `AMZN` 같은 초고유동성 종목으로 확장한다.

## 4. 전략 규칙 명세

### 4.1 세션 규칙

- 프리마켓과 애프터마켓 데이터는 저장할 수는 있지만 신호 계산에는 사용하지 않는다.
- 당일 정규장 시작 후 `09:30~09:45` 첫 15분봉이 완성되기 전에는 진입하지 않는다.
- 신규 진입 가능 시간은 `09:45:00`부터 `11:00:00 ET`까지다.
- `11:00:00 ET` 이후 미체결 주문은 모두 취소한다.
- `11:00:00 ET`에 포지션이 남아 있으면 시장가 또는 공격적 지정가로 강제 청산한다.

### 4.2 기준선 정의

당일 첫 15분봉을 `OR15`라고 한다.

- `or_high = high(09:30~09:45)`
- `or_low = low(09:30~09:45)`

V1은 롱만 사용하므로 `or_high`만 진입 트리거에 사용한다.

### 4.3 몸통 돌파 정의

5분봉에서 아래 조건을 동시에 만족해야 "몸통 돌파"로 인정한다.

- 해당 5분봉의 `close > or_high`
- 해당 5분봉의 `open <= or_high`

즉, 윗꼬리만 넘긴 것은 무시하고 종가 기준으로 돌파가 확정되어야 한다.

### 4.4 bullish FVG 정의

FVG는 연속된 3개 5분봉 `A, B, C`로 정의한다.

- `A = i-2`, `B = i-1`, `C = i`
- bullish FVG 조건: `low(C) > high(A)`
- FVG 구간:
  - `fvg_low = high(A)`
  - `fvg_high = low(C)`

추가 필터:

- `C.close > or_high`
- `fvg_high - fvg_low >= max(0.05, close(C) * 0.0005)`

위 필터는 너무 작은 갭을 제거하기 위한 것이다.

### 4.5 매수 setup 정의

아래 순서를 모두 만족해야 setup이 성립한다.

1. 첫 15분봉 완성
2. 5분봉에서 `or_high` 몸통 돌파 발생
3. 같은 시점 또는 직후 연속 구조에서 bullish FVG 형성
4. 가격이 다시 FVG 구간으로 되돌아옴

V1에서는 첫 유효 setup만 사용한다.

### 4.6 진입 가격 정의

영상 표현은 "FVG 구간으로 들어오면 매수"에 가깝다. 자동화에서는 다음처럼 고정한다.

- 기본 진입가: `fvg_high`
- 체결 방식: `buy limit @ fvg_high`
- 유효 시간: setup 성립 후 `11:00:00 ET`까지

즉, 되돌림이 시작되어 FVG 상단에 닿을 때 가장 먼저 진입한다.

대안으로 midpoint 진입도 가능하지만, V1은 영상 해석에 더 가까운 `첫 터치`를 사용한다.

### 4.7 손절 가격 정의

영상의 "FVG 형성된 캔들의 이전 캔들 저점"은 자동화 시 애매하므로 아래처럼 고정한다.

- FVG가 `A, B, C`로 형성되었을 때
- 손절가: `stop_price = low(A)`

보조 필터:

- `entry_price - stop_price`가 `entry_price`의 `0.15%` 미만이면 스킵
- `entry_price - stop_price`가 `entry_price`의 `1.20%` 초과면 스킵

너무 타이트한 손절과 너무 넓은 손절을 모두 제거한다.

### 4.8 익절 가격 정의

기본은 손익비 `1:2`다.

- `risk_per_share = entry_price - stop_price`
- `take_profit = entry_price + 2 * risk_per_share`

옵션:

- 공격형 모드: `3R`
- 분할 청산 모드:
  - `50% @ 2R`
  - 나머지 `50%`는 진입가로 스탑 이동 후 `3R` 또는 `11:00 ET` 강제청산

V1은 단순성을 위해 `전량 2R 청산`을 권장한다.

### 4.9 일일 거래 제한

- 하루 최대 진입: `1회`
- 손절이 나가면 당일 종료
- 익절이 나가면 당일 종료
- setup이 없으면 무거래

## 5. 리스크 관리

### 5.1 포지션 크기

계좌 기준 리스크 고정 모델을 사용한다.

- `account_risk_pct = 0.25%`
- `max_risk_dollars = equity * account_risk_pct`
- `qty = floor(max_risk_dollars / (entry_price - stop_price))`

추가 제한:

- 총 매수 금액이 계좌의 `20%`를 넘으면 스킵 또는 수량 축소
- 수량이 1주 미만이면 스킵

### 5.2 거래 금지 조건

아래 중 하나라도 만족하면 당일 거래하지 않는다.

- 데이터 누락
- 첫 15분봉 미완성
- 스프레드 과대
- 주문 가능 수량 0
- `LULD`, 거래정지, 급격한 호가 이상
- 브로커 상태 비정상

선택 규칙:

- `FOMC`, `CPI`, `NFP` 같은 대형 이벤트 발표일은 스킵 가능
- 개별 종목 확장 시 실적 발표일은 스킵

## 6. 시스템 아키텍처

### 6.1 컴포넌트 구성

1. `MarketDataAdapter`
- 1분봉 과거 데이터 조회
- 실시간 1분봉/체결 스트림 구독
- 시장 상태 조회

2. `BrokerAdapter`
- 주문 제출
- 주문 정정/취소
- 체결 내역 조회
- 포지션/현금/오류 상태 조회

3. `BarBuilder`
- 1분봉을 받아 5분봉, 15분봉 생성
- 세션 경계와 시간대 처리

4. `SignalEngine`
- opening range 계산
- body breakout 판정
- FVG 탐지
- entry/stop/target 산출

5. `RiskEngine`
- 수량 계산
- 거래 가능 여부 판단
- 일일 제한 관리

6. `ExecutionEngine`
- 지정가 진입 주문 제출
- 스탑/익절 주문 배치
- 강제청산 처리

7. `StateStore`
- 당일 상태 저장
- 재시작 시 복구
- 주문/포지션 id 추적

8. `Journal`
- 모든 신호, 주문, 체결, 취소, 스킵 이유를 저장

9. `Notifier`
- Slack/Telegram/Discord 알림

### 6.2 상태 머신

전략 상태는 아래처럼 단순하게 유지한다.

- `IDLE`
- `BUILD_OR15`
- `WAIT_BREAKOUT`
- `WAIT_RETRACE`
- `ENTRY_PENDING`
- `POSITION_OPEN`
- `DONE`

전이 규칙:

- 장 시작 직후 `BUILD_OR15`
- `09:45` 이후 `WAIT_BREAKOUT`
- breakout + FVG 발생 시 `WAIT_RETRACE`
- 진입 주문 제출 시 `ENTRY_PENDING`
- 체결 시 `POSITION_OPEN`
- 손절/익절/11:00 강제청산 시 `DONE`

## 8. 체결 모델

백테스트와 실거래 차이를 줄이려면 체결 가정이 중요하다.

V1 체결 규칙:

- 지정가 매수 주문은 이후 바의 저가가 `entry_price` 이하일 때 체결된 것으로 본다.
- 손절 주문은 이후 바의 저가가 `stop_price` 이하일 때 체결된 것으로 본다.
- 익절 주문은 이후 바의 고가가 `take_profit` 이상일 때 체결된 것으로 본다.
- 같은 바에서 손절과 익절이 모두 닿으면 보수적으로 `손절 우선` 처리한다.

슬리피지 가정:

- `SPY`: 1틱 또는 `0.01%` 중 큰 값
- 수수료는 브로커 실제 정책 반영

## 10. 브로커/데이터 설계 원칙

### 10.1 브로커 분리

실행 브로커는 전략과 반드시 분리한다.

이유:

- 백테스트와 모의투자는 Alpaca가 편할 수 있다.
- 실거래는 IBKR 또는 국내 증권사 해외주식 API가 더 현실적일 수 있다.
- 브로커를 바꿔도 `SignalEngine`과 `RiskEngine`은 그대로 유지되어야 한다.

### 10.2 데이터 분리

데이터 공급원과 주문 공급원을 분리할 수 있게 설계한다.

예:

- 데이터: Alpaca / Polygon
- 주문: Alpaca / IBKR / 한국투자 해외주식 API

## 14. 열린 쟁점

영상 기반 전략이라 아래는 반드시 추가 검증이 필요하다.

- FVG 첫 터치가 최선인지 midpoint가 더 나은지
- `2R`와 `3R` 중 실제 기대값이 더 높은 쪽
- `SPY` 외 종목 확장 시 성과 저하 여부
- 11:00 강제청산이 유리한지, 12:00까지 허용이 나은지
- 큰 경제 이벤트일 회피가 성과를 개선하는지

## 15. 최종 권장안

처음 구현은 아래로 고정한다.

- 종목: `SPY`
- 시간: `09:30~11:00 ET`
- OR: 첫 `15분봉`
- 신호: `ORH 종가 돌파 + bullish FVG`
- 진입: `FVG 상단 첫 터치`
- 손절: `FVG의 A봉 저점`
- 익절: `2R`
- 거래 수: `하루 1회`
- 리스크: `계좌의 0.25%`

이 조합이 가장 단순하고, 영상 규칙을 덜 왜곡하면서도 자동화 가능한 형태다.
